Requirements

  • Text Editing for source code.
  • Upload/download of files.
  • Compiling supplied source code.
  • Execution console applications
  • For Java, execution of GUI applications.
  • For Java, download compiled class/jar files.
  • For C++, compile and download object/executable files.
  • Test suite creation for automatic assignment marking.
  • Test Running for development.
  • Assignment distribution.
  • Assignment submission.
  • Postsubmission testing, test reports.
  • Reviewing and commenting on submissions
  • Collecting marks
  • Returning assignments with comments, test reports, and marks to students.

Constraints and considerations

  • For console applications, the users (students, instractors, markers) must require no client-side tool other than a web browser running HTML5 / EcmaScript 5 / CSS3.
  • Languages supported should include C++ (gcc) and Java.
  • Additional languages (e.g. Python) should be fairly easy to add.
  • Integration with the Teaching Machine and similar tools would be nice. (TM currently requires Java Plugin on the client.)
  • Security is a very important consideration. We are executing arbitrary code on the server side. Code must be sandboxed in some reasonable way.
