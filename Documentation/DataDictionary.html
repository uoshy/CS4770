Add
When a User adds data to the system the User inputs data to be saved by the system.
Archive Files
A collection of Object Files. The collection can be used for easier bundling of multiple Files within a single File. It can also be referenced by Executable Files at runtime.
Assignment
A method of evaluation for a Course. Each Assignment is linked with one Course and each Course may have many Assignments.
Assignment Solutions
A set of Files supplied by the Instructor which are the proper solutions to a particular Assignment.
Assignment Submission
A set of Files created, and owned, by a Student. The Assignment Submission corresponds to exactly one Assignment. Students may have the option of creating multiple Assignment Submissions for a single Assignment.
Choose
A User chooses an element by selecting it from a group of other options.
Comments
Additional Comments for an Assignment Submission given by the Marker. These Comments can contain information on why a certain Grade was given or how to improve later Assignment Submissions.
Compiler
A program that recieves Source Code Files and turns them into Object Files or Executable Files. The Compiler may give a Compiler Message if the input Source Code Files have errors, or for other reasons.
Compiler Message
Text that describes the operation of a Compiler. Can supply warnings or errors related to input Source Code Files.
Console
A program that runs other programs within it. A Console-based Executable File can be ran and interacted with using a Console. Also known as a terminal.
Course
An offering of a Course in specific semester. Each course has one or more Instructors and 1 or more Students.
Enter
A User enters information by inputting it into the system.
Executable
A program that runs on a computer. Usually it will receive input and give output. These can be Console-based applications or graphical in nature.
File
A collection of data stored on a computer. This could be a text file, binary file, source code file, etc. A User can upload and download files to and from the system.
Grade
A marking on the quality of an Assignment Submission. Usually a numerical value of 100.
Inform
The system informs a User of something by displaying information to them.
Instructor
An Instructor of a Course.
Marker
An assistant for the Instructor of a Course who is responsible for marking Students' Assignment Submissions. They give Students Grades and possibly Comments.
Save
When the system Saves a file, it stores it in its memory such that it is persistent and can be accessed again at a later time.
Select
A User selects an element by clicking on it with the left mouse button.
Object Files
A compiled Source Code File returned by the Compiler. It more closely resembles machine language than the verbose programming language of a Source Code File
Prompt
The system prompts a User by requesting information from them.
Source Code File
A type of File containing programming code. These files are given to a Compiler program to be turned into Object Files and eventually Executable Files.
Student
A Student enrolled in a particular Course. They use the system to create Files and Assignment Submissions. They receive Grades for their Assignment Submissions.
Test Report
An optional report returned to a Student after an Assignment Submission has been submitted. The report reflects the running of a Test Suite on the Assingment Submission, as specified by the Course's Instructor.
Test Suite
A set of programs which automatically test the functionality of a Student's Assignment Submission when it is submitted. An Instructor can supply or create theTest Suite for a particular Assignment.
Users
A User of the software system/website. This may be a Student, Instructor, or Marker.
