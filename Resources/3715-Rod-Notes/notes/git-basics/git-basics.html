GIT

  • Distributed reversion system
  • Created by Linus Torvalds to manage Linux development
  • Can be used to manage any collection of files
  • Very popular (e.g., github)
  • Used for individual revision control
  • Used for distributed collaboration

The following notes will abreviate repository as repo.

Identity and standard configuration

Before using git you should create an identity. The is only required if you plan to use a remote repo. The commands are: 

git config --global user.name "Your Real Name"
git config --global user.email real@email.com

The following ensures typical usage will work at MUN. 

git config --global http.postBuffer 10485760 # 10 megabytes
git config --global push.default matching

Cloning the course repo

The remote repo for 3715 is cloned with: 

% cd  # change to home directory
% git clone https://www.cs.mun.ca/rod-w15-git/3715.git

The above repo is accessed using https, the secure version of http. In this case, a user name and password is required. 

% git clone https://www.cs.mun.ca/rod-w15-git/3715.git

Notice that the password is not echoed. There is a way to avoid typing your password all the time.

Getting started with assignment 1

An example git repo with the commands:

  • git init
  • git add file
  • git commit -m "the log message"

A git repo is only created on the local computer.

A git repo for assignment 1 can be created with: 

% cd
% mkdir 3715-a1
% cd 3715-a1
% # copy the sample html file for submission
% cp ~/3715/git/a1/sample-assign1.html assign1.html
% cp -r ~/3715/git/spark-example exer6
% cp -r ~/3715/git/spark-example exer7
% git init
% git add assign1.html exer6 exer7
% git commit -m "starting assignment 1"
% git status

GIT commands

A summary of git commands from the example are:

git init
Creates an empty repo or resets an existing repo.
git status
Displays files added to the index, untracked files, and files tracked with changes. The index stores the files waiting to be commited.
git add
Adds the files/directories specified in the command to the index. The current contents of the files are added. The entire contents of the directory is added. If a file changes, it should be added again.
git commit
Stores all the files in the index to the repo, a tree object and a commit object are also stored. The tree objects contains the root of all the files in the current snap shot. The -m allows a log message to be specifed.
git log
Displays all of the commit log entries for the current branch. The pager can be disabled with --no-pager

Pushing to a remote repo at MUN

The local repo can be pushed to a remote repo. In this course, the repo can be pushed with: 

% git remote add origin https://www.cs.mun.ca/rod-w15-git/3715/rod/a1.git
% git push --all origin -u  # only require for the first push

The last rod in the URL should be replaced by your labnet id. For example, if your id is foo123 then the URL would be: 

%  git remote add origin https://www.cs.mun.ca/rod-w15-git/3715/foo123/a1.git

Pushing to the repo is done with: 

git push

after the first push.
