color and background color

The page is styled with:

examples/colors.css

div.color-slide { background : rgb(50%,100%,50%); }
p { background : orange; }
li:first-child { color : #00F; }
li { background : gray; color : #FFD500; }
span.green {background : rgb(255,0,255); color : rgb(0,255,0); }

The color of the text in an element is specified with the color property. The color of rectangular background region of an element is specified with the background property.

Colors can be specified with:

  • One of the following: aqua, black, blue, fuchsia, gray, green, lime, maroon, navy, olive, orange, purple, red, silver, teal, white, yellow.
  • rgb(r%,g%,b%) or rgb(0-255,0-255,0-255)
  • #rgb or #rrggbb where r, g, b is a hex digit

Background images can be used to produce good looking pages, however they can be used to produce ugly pages as well. The background images are affected by the following properties:

  • background-repeat: repeat, repeat-x, repeat-y, no-repeat, inherit
  • background-position: left, center, right, percentage, ...

fonts

Fonts are affected by the following properties:

font-family
Specifies a list of possible font families. The font-family can be specific or generic. The generic fonts are: serif, sans-serif, monospace, cursive. A specific font is a font name like 'New Century Schoolbook', or Times.
font-size
The size can be absolute (e.g,. 12pt, 20mm) or relative (e.g., 50%, 1.5em). The follow can also be used: xx-small, x-small, small, medium, large, x-large, xx-large, smaller, larger.
font-weight
The weights include: 100, 200 to 900, bold, bolder, lighter, and normal.
font-style
The styles are: italic, oblique, and normal. Not all font families have all styles.
font-variant
The only variant besides normal is small-caps.

text alignment

Text can be aligned in a containing block element with the text-align property with values of left, center, right, justify, and inherit. Consider:

examples/align.css

td.p1 {text-align: left; border:thin dashed red;}
td.p2 {text-align: center; border:thin solid blue;}
td.p3 {text-align: right; border:thin dotted rgb(200,100,50);}
td.p4 {text-align: justify; border:thin groove yellow;}
table { font-size : inherit; }
All of this text will be left aligned. The right edge will be ragged. The text here is centered. This alignment replaces the center tag. Right aligned text will have a ragged left edge. Justified text will attempt to have straight left and right edges. However the spacing between the words can look bad.

indenting first line

The first line of many paragraphs are indented. The text-indent property controls the amount of the indent. Possible values are: length, percentage, and inherit. Consider,

examples/text-indent.css

div.indent { margin-left: 10%; }
p.i1 { text-indent: 3em; width: 75%; border: thin red solid; }
p.i2 { text-indent: -0.5in; width: 75%; border: thin blue solid; }
p.i3 { text-indent: 1in; width: 75%; border: thin green solid; }

This paragraph is indented by 3em, it is always a good idea to use relative lenghts.

Negative values can be used to create another common formatting style. Many of the style parameters found in word processors are also present in CSS.

A fixed measure of 1 inch (in) can also be used to set a standard indent amount. Millimeters (mm) are another possible unit.

transforming text

Headings are often capitalized or in all uppercase. The text-transform property can be used to ensure that a head follows a particular style. The possible transformations are: uppercase, lowercase, capitalize, none, and inherit.

h2 { text-transform: lowercase; }

whitespace

The web browser normally ignores white space characters when rendering a web page. Whitespace is not ignored inside of a pre element. The treatment of whitespace is controlled by the white-space property. Possible values are: normal, nowrap, pre, pre-wrap, pre-line, and inherit. An example of controlling how whitespace is handled is:

examples/white-space.css

p.pre1 { white-space : pre; }

The paragraph acts like a pre element. The newlines are not ignored.

box model

A box with margins, borders, and padding is defined for all content (e.g., p, span, h1, ...). The top, right, bottom and left sides of the margins, borders, and padding can be individually specified.

The box properties for this paragraph are:
margin: 0.5in 20% 0.25in 10%;
padding: 1.5em;
border: 4px solid black;
background: rgb(200,200,220);

  • The margins are see through.
  • The background setting covers the padding area.
  • The padding is 1.5em for all four sides.
  • Individual parts are specified with x-top, x-right, x-bottom, and x-left where x is margin, padding, and border.
  • The hover changes the margin to margin: 0.25in 5% 0.25in 5%.

box model layout

The box model affects the layout depending on the type of HTML element. The element types are:

non-replaced element
The text content contained in p, li, ... elements.
replaced element
An element that is replaced by external content during rendering (e.g., img, input, ...).
block-level
Elements that occupy the full horizontal space and stack vertically in left-to-right, top-to-bottom languages.
inline
Content is placed on a line when rendered (i.e., all text is placed on a line, so are images).

Stacking blocks

Block elements obey margins, padding, and borders. Consider the following style sheet:

box model: width and height

The height and width can also be specified for block elements, but not inline elements. Instead of specifying the margins, the last example can be specified with the width property.

examples/para-width.css

p.p1 {
    margin-bottom : 30px;
    margin-left: 10%; width: 70%;
    border : 2px dashed red;
}
p.p2 {
    margin-top : 20px; margin-bottom : 30px;
    margin-left: 20%; width: 70%;
    border : 4px solid blue;
}
p.p3 {
    margin-top : 15px;
    margin-left: 30%; width : 40%;
    border : 1px dotted rgb(200,100,50);
}

A width of 70% and a left margin of 10% creates this block. The padding is inherited. The margins between paragraphs collapse to the maximum of the two adjacent blocks.

The margin has pushed this paragraph further to the right. Its width is also 70%.

This paragraph is centered with a width of 40%. There is no other content to the left or right of a block element.

The total size of a block element is the sum of the margins, border, padding, and width. The width specifies only the size of the content, not the entire element.

display and visiblity

The display property controls how an element is rendered. The default value for most HTML elements should only be changed in extreme circumstances. The most important values are:

inline
Placed on a line (e.g,. text).
block
Stacked vertically, disallows any other block element horizontally.
none
The element is not rendered, the content is not considered part of the document.

The visiblity property can be used to hide elements from view. Unlike, display:none property the content still occupies space. The values for visiblity are: visible, hidden, and inherit.

position property

Elements can be shifted from their normal position, or completed removed and placed anywhere on the page. The position property controls where an element is placed. Its values are:

static
placed where it normally would be.
relative
the element is shifted, but the orginal space is maintained.
absolute
the element is placed at on absolute position relative to its containing element.
fixed
the element is placed relative to the browers window.

The properties top, bottom, left, and right control where the element is positioned.
