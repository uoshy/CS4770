Selector in CSS

Cascading Style Sheet associates rendering properties with HTML elements using a set of rules. A rule is composed of a selector followed by a block of rendering properties. For example: 

h1 { font-size: 2em; color : blue; }
p  em { background: red; }

The selector in the first rule identifies all h1 elements. The second rule selects any em element inside of a p element.

Html-tree mapping

  • Text in the document without any tags become one or more text nodes. Text nodes are always leaf nodes.
  • Every element in a HTML document is a node in the tree.
  • The contents of an element, X, will be either children of X or descendants of X. Children of X are nodes that have X as their immediate enclosing element.
  • The html element is the root of the tree.

tree paths and selectors

A path in the tree can be identified by listing the nodes that are encountered along the path. Some possible paths include:

  1. html head title
  2. body p em
  3. body em
  4. p em

A CSS selector is a partial path in the document tree that decends towards the leaves. The selector, body em, selects any em nodes that are decendants of a body node. Notice that only elements can be selected, text nodes cannot be selected.

A more efficient way of considering selectors is to start at the last element, and ask if the preceeding element is an ancestor.

css2 selector summary

*
Matches any element
E
Matches any E element
E F
Matches any F element that is a descendant of an E element.
E > F
Matches any F element that is a child of an element E.
E + F
Matches any F element immediately preceded by a sibling element E.
E[attr]
Matches any E element with the "attr" attribute.
E[attr="a_value"]
Matches any E element whose "attr" attribute value is exactly equal to "a_value".
E[attr~="a_value"]
Matches any E element whose "attr" attribute value is a list of space-separated values, one of which is exactly equal to "a_value".
E.class-name
Matches any element using E[class~="class-name"]
.class-name
Matches *.class-name
E#id-name
Matches E[id="id-name"]
#id-name
Matches *#id-name

pseduo classes

The following pseduo classes can be used to identify special elements.

:first-child
select the element only if is the first element of a parent
:only-child
select the element only if is the only element of a parent
:nth-child(n)
select the nth child element, (e.g., nth-child(odd), nth-child(even), nth-child(5), nth-child(3n+0) )
:first-letter
selects the first letter of a block element
:first-line
selects the first rendered line of a block element (e.g., p:first-line, first line of p)
:before
allows content to be inserted before an element
:after
allows content to be inserted after an element
:link
select an anchor containing a unvisited link
:visited
selects an anchor that have been visited
:focus
an element is selected only if it has received focus
:active
an anchor that has be pushed, but not yet "clicked"
:hover
an element where the mouse is over it
:disabled
select an element that is disabled
:enabled
select an element that is enabled
:checked
select an element that is checked
:not(p)
select an element where p does not occur
